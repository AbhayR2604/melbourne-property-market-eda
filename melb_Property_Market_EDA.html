Data Preprocessing

To ensure the property_transaction_victoria.csv dataset is suitable for analysis, several preprocessing steps were applied to address anomalies and ensure consistency, as required by the assignment specification:

  1. Filtering 9-Digit IDs: Records were filtered to retain only those with a 9-digit numeric id using str_detect(id, "^\\d{9}$"). This ensures compliance with the dataset specification, removing invalid or malformed IDs to maintain transaction uniqueness and data integrity.

  2. Converting Price to Numeric: The price column was cleaned by removing dollar signs, commas, and spaces (str_remove_all(price, "[$,\\s]")) and converted to numeric (as.numeric(price)). This enables accurate numerical computations, such as correlations and predictions, by ensuring consistent data types.

  3. Standardizing Textual Fields: The suburb, property_type, and listing_company_name columns were standardized by converting to lowercase, removing punctuation, replacing “&” with “and”, and applying title case (e.g., str_to_lower() %>% str_replace_all("[[:punct:]]", "") %>% str_to_title()). This reduces variability (e.g., “St Kilda” vs. “st-kilda”) for reliable grouping and filtering.

  4. Removing Duplicates: Duplicate records were removed based on id using distinct(id, .keep_all = TRUE). This ensures each transaction is unique, preventing overcounting in analyses like transaction counts.

These steps address data quality issues, ensuring the dataset is clean, consistent, and ready for accurate exploratory data analysis, as emphasized in the assignment’s focus on effective data wrangling.

To identify properties with the highest capital gain, we grouped the dataset by property address and considered only those sold more than once. We calculated the difference between the first and last recorded sale prices and measured the time span between these transactions. Properties with a duration greater than five years were excluded. The top five properties by capital gain, where all sales occurred within a five-year window, were selected for final reporting.

Grouping by address and gathering the details of the first and last sale

property_gain_data <- multiple_sales_data %>%
  group_by(full_address) %>%
  filter(n() >= 2) %>%  # Keeping only properties sold more than once
  summarise(
    first_sale_price = first(price),
    last_sale_price = last(price),
    capital_gain = last(price) - first(price),
    first_sale_date = first(sold_date),
    last_sale_date = last(sold_date),
    duration = as.numeric(difftime(last_sale_date, first_sale_date, units = "days")) / 365,
    .groups = "drop"
  ) %>%
  filter(duration <= 5) %>%  # Only keep sales within 5-year window
  arrange(desc(capital_gain)) %>%
  slice_head(n = 5)  # Top 5 properties by capital gain

We trained a linear regression model using historical sales data for 4-bedroom, 2-bathroom houses across the six target suburbs. The model used suburb, year, and month as predictors of property price. We then applied this model to forecast expected prices for September 2025 in each suburb, based solely on patterns observed in the existing dataset.
